# CNNVD 下拉框速查表

用于 CNNVD 第 1 页必填下拉框的快速判断。遇到下拉框时，先查这份表，再执行选择。

目标：

- 少展开
- 少判断
- 少重复交互
- 优先一次选对

---

## 一、使用规则

只处理这三个必填下拉框：

1. `漏洞类型`
2. `漏洞自评级`
3. `受影响实体分类`

执行顺序：

1. 先运行一次 `prepare_form_context.py`，直接读取 `dropdown_plan.vuln_type_path`、`dropdown_plan.risk_level`、`dropdown_plan.affected_entity_category`
2. 遇到下拉框先查本表
3. 能直接命中就直接选
4. 不能命中再按兜底规则处理

注意：

- 第 2 页和第 3 页不要重新提取数据。
- 非必填下拉框不处理。
- 如果动态出现新的必填下拉框，能选 `其他` 就选 `其他`。

### 3. 选择原则

- 优先命中上表，不临场发挥。
- `文件包含` 一律归到 `路径遍历`。
- `文件上传` 当前统一归到 `后置链接`，方便快速落类。
- 明确不属于现有层级时，选 `其他`。

## 五、兜底规则

### 1. 漏洞类型兜底

- 能归到明确子类就不要选 `其他`
- 完全无法对应时才选 `其他`

### 2. 受影响实体分类兜底

- 能明确归到具体平台类目时，不要选 `其他`
- 网站/CMS/博客类优先判断 `建站系统`
- 通用业务站点优先判断 `web应用`
- 服务端组件优先判断 `中间件`
- 仍然无法判断时，选 `其他`

### 3. 交互兜底

- 快照里看不到下拉选项时，用 `evaluate_script` 点击下拉 DOM
- 不要因为一个下拉框反复 `take_snapshot`
- 级联下拉必须点击最终叶子选项前面的圆圈/单选按钮完成选择，不要只点击文字
- 不要按 Escape 关闭；Escape 可能导致选项没有选中

示例：

```javascript
const target = Array.from(document.querySelectorAll('.el-cascader-node, .el-select-dropdown__item'))
  .find((el) => el.innerText.trim() === '目标选项');
const radio = target?.querySelector('.el-radio, .el-radio__input, input[type="radio"]');
(radio || target)?.click();
```

## 六、使用方式

实际操作时按这个顺序：

1. 看 `extract_vuln_data.py` 输出
2. 先查本文件

不要在页面上边看边猜。
