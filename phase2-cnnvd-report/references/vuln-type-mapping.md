# CNNVD 漏洞类型映射

## 一、漏洞类型映射表

| 数据值 | CNNVD 级联路径 |
|-------|--------------|
| 命令执行/命令注入 | 代码问题 → 输入验证错误 → 注入 → 命令注入 |
| 越界写入/堆溢出 | 代码问题 → 输入验证错误 → 缓冲区错误 |
| SQL注入 | 代码问题 → 输入验证错误 → 注入 → SQL注入 |
| XSS/跨站脚本 | 代码问题 → 输入验证错误 → 注入 → 跨站脚本 |
| 代码执行/代码注入 | 代码问题 → 输入验证错误 → 注入 → 代码注入 |
| 路径遍历 | 代码问题 → 输入验证错误 → 路径遍历 |
| 文件包含 | 代码问题 → 输入验证错误 → 路径遍历 |
| CSRF | 代码问题 → 输入验证错误 → 跨站请求伪造 |

## 二、漏洞类型层级结构

```
代码问题
├── 输入验证错误
│   ├── 注入
│   │   ├── 代码注入
│   │   ├── 跨站脚本
│   │   ├── 命令注入
│   │   ├── 格式化字符串错误
│   │   └── SQL注入
│   ├── 跨站请求伪造
│   ├── 缓冲区错误    ← 越界写入/堆溢出选此项
│   ├── 后置链接
│   └── 路径遍历
├── 授权问题
├── 竞争条件问题
├── 处理逻辑错误
├── 数字错误
├── 未声明问题
├── 加密问题
├── 数据转换问题
└── 资源管理错误
配置错误
其他
环境问题
```

## 三、级联选择操作

漏洞类型是必填级联下拉菜单，只需要选择一次。优先用 `evaluate_script` 查找并点击 Element UI 下拉项，减少 `take_snapshot`、uid 点击和重复交互。

**示例（命令执行）**：
1. 点击漏洞类型下拉框
2. 第一级：点击"代码问题"
3. 第二级：展开"输入验证错误"
4. 第三级：展开"注入"
5. 第四级：点击最终叶子选项"命令注入"前面的圆圈/单选按钮，让级联下拉自动收起

**示例（文件包含）**：
1. 点击漏洞类型下拉框
2. 第一级：点击"代码问题"
3. 第二级：展开"输入验证错误"
4. 第三级：点击最终叶子选项"路径遍历"前面的圆圈/单选按钮，让级联下拉自动收起

不要按 Escape 关闭级联下拉。Escape 可能导致最后一个选项没有真正选中。也不要只点击文字，优先点击选项前面的圆圈/单选按钮。

如果 `take_snapshot` 只显示 listbox 但没有选项，使用页面脚本：

```javascript
const target = Array.from(document.querySelectorAll('.el-cascader-node, .el-select-dropdown__item'))
  .find((el) => el.innerText.trim() === '目标选项');
const radio = target?.querySelector('.el-radio, .el-radio__input, input[type="radio"]');
(radio || target)?.click();
```
